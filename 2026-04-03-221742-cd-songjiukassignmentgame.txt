  지금 Ease.OutBack이 이미 "살짝 커졌다가 줄어드는" 효과를 주긴 하는데, 정확히

  ---
  힌트

  DOTween에서 중간 값을 거쳐가는 애니메이션은 DOTween.Sequence()를 써요.

  Sequence는 두 개의 트윈을 순서대로 이어붙이는 방식이에요:

  1단계: 0 → originScale * 1.5f  (빠르게 커짐)
  2단계: originScale * 1.5f → originScale  (살짝 줄어듦)

  구조 힌트

  DOTween.Sequence()
      .Append( /* 0 → 1.5배 트윈 */ )
      .Append( /* 1.5배 → originScale 트윈 */ );

  - Append의 첫 번째 인자: _zone.transform.DOScale(목표스케일, 시간)
  - 첫 번째 트윈의 목표 스케일 → originScale * 1.5f
  - 두 번째 트윈의 목표 스케일 → originScale
  - 시작을 0으로 만들려면 첫 번째 트윈에 .From(Vector3.zero) 추가

❯ 이거 OnPurchaseComplete이게 연속으로 들어와서 바로 machine으로

⏺ Read 1 file (ctrl+o to expand)

⏺ 맞아요, 정확히 그 이유예요.

  타이밍을 보면:

  0.00초: remainCost-- (1→0), money1 DOJump 시작 (0.1초 소요)
  0.05초: 루프 재실행 → remainCost가 이미 0 → while 조건 통과 안 함  ← OK

  근데 remainCost가 2 이상이면?
  0.00초: remainCost-- (2→1), money1 DOJump 시작
  0.05초: remainCost-- (1→0), money2 DOJump 시작
  0.10초: money1 OnComplete → remainCost == 0 → OnPurchaseComplete() 호출!
  0.15초: money2 OnComplete → remainCost == 0 → OnPurchaseComplete() 또 호출!

  remainCost--는 0.05초마다 즉시 실행되는데, OnPurchaseComplete 체크는 DOJump 
  완료 후(0.1초) 에 실행돼서 여러 개가 겹쳐요.

  ---
  힌트: 가장 간단한 해결은 이미 완료됐는지 체크하는 플래그 하나 추가하는 거예요.

  bool isCompleted = false;

  // OnComplete 안에서
  if (remainCost == 0 && !isCompleted)
  {
      isCompleted = true;
      OnPurchaseComplete();
  }

  직접 고쳐봐요!
