  1. 어떤 NPC가 필요한가?

  ┌──────────┬─────────────────────────────────────┬───────────┐
  │   NPC    │                역할                 │ 이동 패턴 │

  │ 보조 NPC │ 수갑 보관소 ↔ 입구 반복 순환        │ 루프      │
  └──────────┴─────────────────────────────────────┴───────────┘

  ---
  2. 설계의 핵심 질문

  "이동 로직"과 "행동 로직"을 분리해야 하나?

  현재 PrisonerController는 이동(AsyncMoveToPosition) + 행동(ReceiveHandCuff
  등)이 한 파일에 있습니다.

  컴포넌트 분리 방식:
  NPCMover.cs        → 어디로 갈지 (Waypoint 이동만 담당)
  [NPC역할]Controller → 무엇을 할지 (행동 상태만 담당)

  분리하면 좋은 이유: 보조 NPC든 수감자든 이동 로직은 같고, 행동만 다르기
  때문이에요.

  4. Waypoint 구조

  DOTween의 DOPath()로 경로 배열을 따라 이동하는 게 가장 깔끔해요:

  // 무한대 모양 waypoint 생성
  Vector3[] GetInfinityPath(Vector2 center, float size, int count = 30)
  {
      var points = new Vector3[count];
      for (int i = 0; i < count; i++)
      {
          float t = (float)i / count * Mathf.PI * 2;
          float x = size * Mathf.Sin(t);
          float y = size * Mathf.Sin(t) * Mathf.Cos(t);
          points[i] = new Vector3(center.x + x, center.y + y, 0);
      }
      return points;
  }

  // 재생
  finger.DOPath(GetInfinityPath(screenCenter, 80f), 3f, PathType.CatmullRom)
        .SetLoops(-1, LoopType.Restart)
        .SetEase(Ease.Linear);
